## Product thesis

Hands-free AI is most valuable when the user is doing something physical: fixing an object, looking at a worksheet, reading a label, following instructions, or moving through an unfamiliar environment. A phone chatbot is useful, but it requires the user to stop, hold the device, frame the scene, and type or tap.

VisionZen explores the next step: a voice-first assistant with visual context from wearable glasses.

---

## Target scenarios

| Scenario | Example |
|---|---|
| Object identification | "Hey Z, what am I holding?" |
| Troubleshooting | "This is leaking. Tell me what to check first." |
| Homework review | "Review this problem and help me understand the next step." |
| How-to guidance | "Walk me through fixing this without doing it for me." |
| Translation | "Translate this label and tell me if there are warnings." |
| Field workflow | "Ok Zen, what's next?" |

---

## Product principles

1. **Explicit activation beats ambient surveillance.** The app can preview locally, but it should not upload frames while idle.
2. **Cost posture must be visible.** AI billing state is part of the UX, not hidden infrastructure.
3. **Grounding matters.** Real-world guidance needs source-backed context, not just visual guesswork.
4. **The assistant should be interruptible.** A field assistant that keeps talking becomes unusable quickly.
5. **Manual fallback is a feature.** If API mode is off, the user should still be able to package a frame and prompt for ChatGPT Pro.

## Why this is recruiter-relevant

This project is not just a demo. It shows:

- product discovery under real device constraints
- mobile UX designed around field use
- wearable SDK integration
- real-time AI state management
- cost containment after a real billing surprise
- privacy-aware documentation choices
- public communication without leaking code
